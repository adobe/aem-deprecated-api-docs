# Deprecation wave with enforcement date June 11

This wave captures the deprecated API packages and classes that Adobe highlights ahead of the **June 11, 2026** enforcement milestone.

[Adobe Experience League: Deprecated and removed features](https://experienceleague.adobe.com/en/docs/experience-manager-cloud-service/content/release-notes/deprecated-removed-features#removed-apis)

In order to validate your application for this deprecation, you are recommended to use the version 1.6.18 of the AEM analyser in your project. See [/docs/aem-analyser-maven-plugin.md](the documentation) how to run this plugin during the build of your AEM application.



## Time table

Adobe describes the following timeline for these deprecated APIs:

- **January 26, 2026**: reminder notifications begin
- **February 26, 2026**: Cloud Manager pipelines pause during Code Quality and can be overridden
- **April 14, 2026**: Cloud Manager pipelines fail during Code Quality
- **June 11, 2026**: environments still using these APIs do not receive critical Adobe release updates

## How To Use This Wave

1. Run the AEM as a Cloud Service SDK Build Analyzer Maven Plugin in the customer project.
2. Identify the affected OSGi bundles for any package or class listed above.
3. Create or update bundle guides under `docs/bundles/` for the bundles that are actually affected.
4. Keep the implementation details in the bundle guide rather than duplicating them in this wave page.

## Notes

- This page is a source-aligned inventory for the June 11, 2026 wave.
- It does not yet include bundle-specific remediation steps.
- If Adobe changes the source table, this page should be updated to stay aligned.
